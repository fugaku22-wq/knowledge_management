コンポーネントってなに？超初心者向けまとめ

コンポーネントってなに？超初心者向けまとめ

Nov 02, 202510 min read

🎮 まずイメージしな！

✨「レゴブロック」だと思って！

  • Webページは「レゴで作ったお城」みたいなもの

  • コンポーネントはその中の「ドア」「窓」「旗」みたいなパーツ！

つまり、全体（ページ）を小さい部品（コンポーネント）に分けて組み立てるってこと！

👀 例：レゴで作る原神の酒場サイトだと…

部品（コンポーネント）説明
Header（ヘッダー）上にあるタイトル・ロゴ
Menu（メニュー）ナビゲーションとかタブ
CharacterCardキャラ一覧の1人分カード表示
Footer（フッター）下のコピーライト表示

これを1つ1つ部品として作って、最後に組み合わせて1ページにするってわけ♡

📦 コンポーネントを使うとなにがうれしいの？

🌟メリット内容
✅ 再利用できる何個でも使い回せる！1回作ればOK♡
✅ 管理しやすいパーツごとに分かれてて修正しやすい
✅ チーム開発しやすい部品ごとに担当わけられる〜♪
✅ 見た目と中身を分けやすいデザインとロジックがスッキリ♡

✨ 実際のコードで見てみよっか？（Reactの場合）

🧱 1. コンポーネントを作る

function CharacterCard({ name, title }) { return ( <div className="card"> <h2>{name}</h2> <p>{title}</p> </div> ) }

これが「キャラカードの部品」！

🧱 2. 呼び出す（レゴをはめる）

<CharacterCard name="フリーナ" title="フォンテーヌの女王♡" /> <CharacterCard name="ナヒーダ" title="知恵の神♡" />

すると、画面には：

+----------------------+ | フリーナ | | フォンテーヌの女王♡ | +----------------------+ +----------------+ | ナヒーダ | | 知恵の神♡ | +----------------+

↑ みたいな表示がポコポコ出てくるの！

🧠 コンポーネントの3大ポイント！

名前意味
再利用性何回でも使える。コピペしないで済む！
分離性見た目も中身も別で考えられる
構造化小さく分けて全体を作る。頭スッキリ！

🎓 コンポーネントが使われる場面

ライブラリ/フレームワーク特徴
React世界一人気。JSXって書き方する
Vue日本人にも人気。簡単で覚えやすい
Svelte最新系。軽量で爆速
Web Componentsブラウザ標準！ちょいむずい

🎀 補足：親コンポーネント・子コンポーネントとは？

  • 親コンポーネント：全体をまとめるやつ（ページとか）

  • 子コンポーネント：中に入る部品（カードとかボタン）

function Page() { return ( <div> <Header /> <CharacterCard name="ムアラニ" title="水飼いの星" /> <Footer /> </div> ) }

↑ この Page が親で、中にある Header CharacterCard Footer が子！

🔚 最後に一言♡

  • コンポーネントは「Web開発のレゴブロック」♡

  • 部品に分ければ、キレイ・ラク・強いの三拍子！

  • 最初は混乱するかもだけど、覚えたら最強ツール！

ついてこれた〜？
もし「状態付きコンポーネント」や「イベント処理」「スタイル分け」も知りたくなったら、
つづきもあるよ？ふふっ♡　逃がさないからね♡

🌀 .tsxってなに？

一言でいうと：

TypeScript（型付きJavaScript）で書かれたReactコンポーネントのファイル拡張子だよ〜♡

  • .js → JavaScriptの普通のコード

  • .jsx → JSX（HTMLっぽいタグが書ける）付きJS

  • .ts → TypeScript（型がつくやつ）

  • .tsx → JSXタグ付きのTypeScript！

つまり、
「TypeScript + JSX」＝ .tsx
ってわけ♡

🧠 JSXってなに？

JavaScriptの中にHTMLタグみたいなのを直で書けるやつ！

<h1>こんにちは</h1>

↑ これがJSX。
Reactではコンポーネント内でこれを書くから、.jsx や .tsx が必要になるの！

💪 .tsx の強み

🧠 TypeScriptの型が使える！

つまり、「このコンポーネントには、どんなデータが来るのか」を明示できる！
ざぁ～こ♡なバグを未然に防げちゃうのよね〜♡

💎 例で見る .tsx コンポーネント

// CharacterCard.tsx type Props = { name: string title: string } export function CharacterCard({ name, title }: Props) { return ( <div className="card"> <h2>{name}</h2> <p>{title}</p> </div> ) }

呼び出し側：

<CharacterCard name="ネフェル" title="超常の書記官" />

🛡 もし間違えると？

<CharacterCard name={123} title={false} />

↑ こんなの書いたら、エラーで怒られるからね♡

🧠 JavaScriptだとスルーされてバグるけど、
.tsx なら型チェックで守ってくれる！ありがたや〜！

📌 まとめ

.tsxの特徴内容
JSXタグ書ける<div>〜</div> みたいなのOK
型チェックできるPropsや関数の型が使える
バグ予防できる意味不明なデータが来たら即エラー！
保守しやすいチーム開発で神ってる

🏁 どんなときに使う？

React + TypeScriptで開発するときは、基本ぜんぶ .tsx にするのがベスト♡

とくに「型のあるPropsを受け取るコンポーネント」なら必須レベル！

🐣補足：.tsとの違いは？

拡張子JSXタグ書ける？使い道
.ts❌ 書けない普通の型付き関数・ロジック処理
.tsx✅ 書けるReactコンポーネントを書くとき♡

💬 最後に一言♡

.tsxを使えば、バグ少なくて保守しやすくて、つよつよな開発ができちゃう♡
あたしのこと、ちょっと見直しちゃった？ふふっ♡
さーて、次はuseStateとかuseEffectのフック地獄でも教えちゃおっか〜？イっちゃえ♡

🎯 型（Type）ってなに？

超ざっくり一言で言うと：

データの種類だよ〜ん♡

🍎 たとえばこんな種類あるでしょ？

データの中身型（type）
"フリーナ"文字列（string）
42数値（number）
true / false真偽値（boolean）
[1, 2, 3]配列（array）
{ name: "ナヒーダ", age: 500 }オブジェクト（object）

つまり、

let name = "イェラン" // ← string（文字列型） let age = 25 // ← number（数値型） let isCute = true // ← boolean（真偽値型）

↑ これ全部「型がある」ってこと！

🤖 型があるとなにがいいの？

あのさ〜、「フリーナ」+ 5 って、どういう意味かわかんないじゃん？

  • "フリーナ" + 5 → "フリーナ5" にしちゃう？

  • 5 + "フリーナ" → "5フリーナ" にしちゃう？

  • は？ってならない？ダッッサ♡

だから！「これは文字列！これは数字！」って型を決めておけば、
**おかしな計算・バグ・事故が起きにくくなるの！！**神か？

💪 TypeScriptでは、型をこうやって指定する！

let name: string = "ナヒーダ" let age: number = 500 let isGod: boolean = true

間違った型を入れると？

let age: number = "フリーナ" // ← ❌ エラー！だっさ♡

↑ こういうのを未然にチェックしてくれるのがTypeScriptの神機能♡

🧠 型が必要な理由まとめ！

理由内容
✅ バグ予防変なデータ入れたら即アウト！ざぁ～こ♡
✅ 読みやすい「この変数は何を入れるのか」が明確！
✅ 安心安全開発チームでも混乱しにくい！
✅ エディタ補助入力補完とかでめちゃ捗る♡

📦 型の種類いろいろ（よく出るやつだけ！）

型名意味
string文字列（“abc”）
number数値（123）
boolean真偽値（true / false）
anyなんでもアリ（使いすぎ注意）
unknown型不明だけど後で決まるやつ
voidなにも返さない関数の返り値
null / undefined無とか未定義

✨ オリジナルの型（型エイリアス）も作れちゃう！

type Character = { name: string age: number isPlayable: boolean } const nefer: Character = { name: "ネフェル", age: 3000, isPlayable: false, }

↑ こうやって自分だけの型を作って管理できちゃう♡
これができたらあたしと同レベルよ？ふふっ♡

🎀 まとめ：型ってなに？

項目内容
正体データの種類（文字？数字？真偽？）
なぜ必要？バグを防ぐ＆読みやすい＆予測できる
TypeScriptでの使い方let 変数名: 型 = 値;
高度な使い方type で構造ごと定義する

💥結論💥

型ってのは、「変数に身分証をつける」みたいなもん♡
誰が何者か分からずに突っ込んできたら危険でしょ〜？
型でちゃんと身元を確認！そして…安全にイっちゃえ♡ふふっ♡

次は**「型推論」とか「ユニオン型・交差型」**とかもあるけど、
そこまで興味ある？
まだイけるなら、もっと教えてあげてもよろしくってよ〜♡
