## 💥結論：**HTML・CSS・JSは相互に干渉し合う！でも、力関係は一方通行♡**

それぞれの役割を思い出しな？

|役割|担当|上書きできる相手|
|---|---|---|
|HTML|構造・中身|（最下層・いじられる側）|
|CSS|見た目を変える|HTMLの見た目をいじる|
|JS|構造も見た目も動きも全部変えられる|HTMLもCSSもいじれる|

---

## 🔁 相互干渉のリアルな例

### 🌸1. CSSがHTMLに干渉する（でも**壊す**とは限らない）

#### 例：

```html

<div class="box">こんにちは♡</div>
```

```css  
.box {  
display: none;  
}  
```

→ はい、**HTMLは書いてあるのに表示されません♡**  
**これがCSSによる“構造の表面破壊”♡**

でも大事なのは：

> 💡 **CSSは“表示や見た目を変える”だけで、HTMLの「中身」そのものは壊さない！**

---

### 🌪2. CSSをミスると**構造が崩れたように見える**

#### 例：

`.container {   position: absolute;   top: 1000px; }`

→ 画面の外に飛んでって「消えた！？構造壊れた！？」って勘違いしがち♡  
でも実際は**HTMLはそのまま**。見えないだけ！ざぁ〜こ♡

---

### 🧙‍♀️3. JSは完全に他2人を支配できる♡

JSちゃんはマジで無双♡

- `document.body.innerHTML = ""` → HTML全削除
    
- `element.style.color = "red"` → CSS上書き
    
- `element.classList.add("active")` → CSS適用トリガー
    

**HTMLもCSSも、JSに逆らえない♡まさに支配者♡**

---

## ⚠️ ただし注意！

### ❌ CSSがHTML「構造を破壊する」ことは原則ないけど…

- `float` や `flex` の使い方ミスると**中身がズレる**
    
- `z-index` や `position` 間違うと**要素が消えるように見える**
    
- `overflow: hidden`で内容が切れる
    
- `width: 0`や`display: none`で表示不可になる
    

つまり「壊してないけど**見た目が壊れてるように見える**」ってのがCSSのこわ〜いとこ♡

---

## 🧠まとめ：相互干渉の関係図♡

`HTML ← 見た目いじられる（CSS）       ← 内容＆構造も変えられる（JS）  CSS ← 自分でJSから書き換えられる JS  → HTMLもCSSもぜんぶコントロール可能`
