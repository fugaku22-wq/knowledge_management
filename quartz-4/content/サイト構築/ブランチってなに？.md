## 🌱 ブランチ（branch）とは？

> 💡**「作業用の分かれ道」＝別ルートでの開発スペース！**

Gitの世界では、**“並列のタイムライン”を作る**のがブランチなの！

---

## 🎮 例えるなら…

> ブランチは「セーブデータの分岐」！

RPGで「ここで選択肢分かれます！」ってときに  
**別のセーブスロットに保存して別ルートを試すでしょ？**

それと同じで：

- `main`ブランチ：ゲーム本編
    
- `feature/new-ui`ブランチ：UI改修の試しプレイ
    
- `fix/bug-22`ブランチ：バグ修正の実験ルート
    

みたいに、**安全に別ルートで開発できるのがブランチ♡**

---

## ✨ ブランチの利点

|メリット|説明|
|---|---|
|✅ 安全に分岐できる|本線（main）を壊さず、新しい試みができる|
|✅ チーム開発に向いてる|各自が別のブランチで作業して、最後に合流（マージ）|
|✅ 比較や戻しも簡単|いつでも切り替えOK。失敗しても main は無事♡|

---

## 🔧 よく使うブランチ操作

|コマンド|意味|
|---|---|
|`git branch`|ブランチの一覧を表示|
|`git branch new-branch`|新しいブランチを作る|
|`git checkout new-branch`|作ったブランチに移動する|
|`git switch -c new-branch`|作って＆移動を一気にやる（最近の書き方）|
|`git merge ブランチ名`|他のブランチの内容を今のブランチに取り込む|
|`git rebase ブランチ名`|履歴を整えて、後ろにつなぎなおす|

---

## 💡 実際の流れ：よくある開発スタイル

`git switch -c feature/button-style （ボタンデザイン変更の作業をする） git add . git commit -m "Update button style" git switch main git merge feature/button-style`

→ こうすれば、**安全に開発して、後で本線（main）に合流**できるの！
