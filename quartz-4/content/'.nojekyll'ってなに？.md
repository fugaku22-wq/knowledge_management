## 💿 `.nojekyll`ってなに？

`.nojekyll` は **GitHub Pages用の特殊なファイル**で、**Jekyll処理をスキップさせるための目印**だよ！

---

## 🤔 え、Jekyllってなによ？

- GitHub Pagesはデフォルトで **「Jekyll」っていう静的サイトジェネレーター** を使ってるの。
    
- でもこのJekyll、**アンダースコア (`_`) で始まるフォルダを無視しちゃう癖**があるのよね〜。  
    　（例：`_assets/` とかが無視されて表示されない💔）
    

---

## 💡 `.nojekyll` を置くとどうなるの？

- Jekyllの処理を完全スルーして、**アップロードしたとおりのファイル構成で表示してくれる**ようになる！
    
- Quartzとか、手動でHTMLとかJSとか置く静的サイトでは、これが**必須レベル**！
    

---

## 🔧 使い方：

- ファイル名：`.nojekyll`
    
- 中身：**空でOK！**
    
- 置き場所：**GitHub Pagesのルートディレクトリ（`main` ブランチ or `docs/` フォルダなど）**
    

`touch .nojekyll git add .nojekyll git commit -m "add .nojekyll to disable Jekyll" git push`

---

## 😏 なにが違うの？

|`.nojekyll` あり|`.nojekyll` なし|
|---|---|
|`_` で始まるフォルダも使える|無視される可能性アリ|
|オレ流のHTML/CSS/JSがそのまま反映|Jekyllに邪魔されるかも|
|QuartzやObsidian出力が安定|予期せぬ表示バグで泣く|

---

## 🎯 結論：

Jekyllの魔の手からサイトを守る、お守りみたいな存在ってわけ♡  
とくに**QuartzでGitHub Pages使うなら、置かないとだっさ♡だよっ！**
