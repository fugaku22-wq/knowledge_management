## 💥 `git init`とは？

あるフォルダを「Gitで管理する場所ですよ～」って宣言するコマンド！

`git init`

って実行すると、そのフォルダ内に `.git` っていう超重要な隠しフォルダが作られるの。

---

## 🧠 何が起きるの？

- `.git/` フォルダが作成される。
    
- これが**Gitが使うデータベースの本体**！  
    コミット履歴、ブランチ情報、設定、全部ここに入る。
    
- 以降、そのフォルダ配下のファイルを `git add` とか `git commit` とかで管理できるようになる！
    

---

## 📦 使う場面（例）

1. 新しいプロジェクト作ったとき：
    
    `mkdir my-project cd my-project git init`
    
2. クローンじゃなくて、**手元のフォルダを後からGitで管理したくなったとき**に便利。
    

---

## 📁 見えない `.git` フォルダって？

こいつが**Gitのすべての記録の司令塔**。  
これ消したらGitの履歴はまっさらになるから要注意！  
（言うなら「記憶喪失」ね♡）

---

## ❌ よくある勘違い

|勘違い|真実|
|---|---|
|`git init` しただけでGitHubにアップされる|されない！GitHubにプッシュするには `git remote add` → `git push` が必要|
|`git init` は一度だけやればいい？|基本はプロジェクトごとに1回でOK。サブモジュールは別だけど。|
|`.git` を削除してもファイルは残る？|ファイルはそのまま。でも**履歴は全部消える！**|
