## 🧠 ざっくり！Gitの全体像：

### これは**「タイムマシン付きのメモ帳」**だッ♡

> **Git＝ファイルの変更履歴をぜんぶ記録して、いつでも戻れる神ツール！**

📚普通のメモ帳：上書きしたら前の内容消えるよね？  
🕰Gitメモ帳：過去の状態が**全部記録されてる**から、  
→「やば！元に戻したい！」ってときもピッと復元できちゃう♡　天才か？

## 🧩 3つのゾーンで管理してるよっ

`[ 作業ディレクトリ ] ← 作業中のファイル（見えてるやつ）        ↓ add [ ステージング ]       ← 変更を記録する準備（提出前）        ↓ commit [ リポジトリ ]         ← 変更の記録（履歴に追加）`

1. **作業ディレクトリ（Working Directory）**  
    　→ 実際にあたしらがファイルをいじってる場所！
    
2. **ステージングエリア（Staging Area）**  
    　→ 「このファイルの変更を記録するよー！」ってマークするところ！  
    　→ `git add` コマンドでここに入れる！
    
3. **リポジトリ（Repository）**  
    　→ 変更が履歴としてガチ保存される場所♡  
    　→ `git commit` で記録される！
    

---

## 🔄 よくある操作の流れ：

`1. ファイルを編集 2. git add ファイル名　→ ステージング！ 3. git commit -m "変更内容"　→ 記録！`

ざっこ♡ ここまでで混乱してたら修行不足～！

## 🌳ブランチってなに？

> ＝ 「別の作業の線路」だよ！

- **mainブランチ**（本線）← 本番用
    
- **feature-〇〇ブランチ** ← 試したい機能、修正用
    

→ **分岐して作業 → 完成したら合体（merge）**って感じ！

## 🛠️ push / pull / cloneって？

|コマンド|意味|よく使う場面|
|---|---|---|
|`git push`|ローカルの変更 → GitHubにアップ|作業終わったら共有！|
|`git pull`|GitHubの最新 → ローカルに取り込み|他の人の変更取り込みたい|
|`git clone`|GitHubの内容を丸ごとコピー|はじめて取得したいとき|

---

## 🧨 間違えたらどうする？

- `git status`：現在の状態確認（ステージングされてる？未保存？）
    
- `git log`：過去のコミット履歴
    
- `git reset`：ステージから外す or 巻き戻す
    
- `git reflog`：ヤバいくらい過去まで巻き戻せる最終兵器♡
