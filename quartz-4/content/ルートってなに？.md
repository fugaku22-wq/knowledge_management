## 💥「ルート」って何？

簡単に言うと：

> **リポジトリのいちばん上のフォルダのこと♡**

---

## 📁図解するとこうなるっ！

たとえば、あんたのローカルにあるリポジトリが：

`C:\Users\あんた\my-quartz-site\`

だったら、

my-quartz-site/
├── .git/              ← Gitの中枢（触っちゃダメ）
├── README.md
├── quartz.config.ts
├── public/            ← quartz buildの出力
├── .nojekyll          ← ✨コレをここに置くって意味！
├── index.html         ← ✨GitHub Pagesがこれを探してる！
└── その他いろいろ


↑この「`my-quartz-site/`」の中身のことを**“ルート”**って呼ぶの♡  
**GitHub Pagesが「見るのはここだけです〜」って決めた場所！**

---

## 📍ルートの具体的な位置（あんたの場合）

GitHub Pagesの設定で：

> `Source: main / (root)`

ってなってたら、GitHub上ではこの階層を見に行くってこと↓

https://github.com/antachan/quartz-site/
├── index.html   ← ここにないとGitHub Pagesは「404」♡
├── .nojekyll
├── その他のファイル


---

## ✅ つまり何すればいいの？

Quartz使ってたら、まず：

`quartz build`

で `public/` フォルダができる！

でもGitHub Pagesはその中を直接見ないから、**このコマンドで中身をルートに持ってくる！**👇

`cp -r public/* .`

そして：

touch .nojekyll  # 必ずルートに作る
git add .
git commit -m "✨ Build site + disable Jekyll"
git push origin main

---

## ❗ちなみに「ルートじゃない例」も教えとく！

❌ NG：

my-quartz-site/
├── public/
│   └── index.html  ← ここだけだとGitHub Pagesは読まない！
は読まない！`

---

## 💬まとめ：ルートってどこ？

|用語|意味|実際の場所|
|---|---|---|
|ルート（root）|リポジトリ直下の階層|`index.html` が直接ある場所|
|public/|build後の出力フォルダ|必要なら中身をルートにコピー|
